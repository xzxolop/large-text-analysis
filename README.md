# large-text-analysis

## Идея приложения

Необходимо провести что-то на подобии кластерного анализа текстов. На вход программе передается слово на английском языке. Например слово data. Далее я получаю слова в контексте с которыми это слово встречается чаще всего:

data
> big 1000 совпадений
> python 200 совпадений

далее я могу раскрыть один из пунктов и получить следующие слова с которыми чаще всего идут совпадения. Например нажав на big: 

data
> big 1000
    > job 500
    > ML 200
    > ...
> python 200

Мой текст разбивается на sentance (предложения) используя sentance tokenization, world tokenization. Каждое предложение будем независимо обрабатывать с помощью TF-IDF индекса, а также используем ElasticSearch. 

Демонстрацию планируется написать с использованием streamlit.

## Запуск проекта

### Запуск веб-версии

Синхронизируем пакеты
```uv sync```

Активируем виртуальное окрежение (для Windows)
```.\.venv\Scripts\activate```

Также можно выбрать интерпретатор виртуального окружения  (по желанию)

Запуск проекта

```uv run uvicorn src.web_app:app --reload```

Откройте в браузере: http://127.0.0.1:8000/pmi

### Запуск тестов

```uv run pytest -v```

### Запуск консольной демонстрации (может не работать)

``` uv run ./src/main.py```

## SQLite

После первой предобработки датасета оригинальные и очищенные предложения
сохраняются в `files/large_text_analysis.sqlite3`.

База содержит две таблицы:

- `sentences` — исходный текст, обработанный текст и связь с документом;
- `metadata` — сведения об исходном CSV и версия алгоритма предобработки.

При следующем запуске приложение проверяет размер и время изменения CSV, а
также версию предобработки. Если данные не изменились, предложения загружаются
из SQLite. Для принудительного пересчета достаточно удалить файл базы данных.
